Getting Started with the libSpGH C++ Shared Library

The Library Compiler in MATLAB® Compiler SDK™ creates C++ shared libraries that can be integrated with applications written using either the C++ Data Array or mwArray APIs. It also generates sample C++ driver code that can be used to integrate and test the generated components. You can use this guide to set up your environment and run your sample driver application.
Note: Sample C++ driver code is only generated if sample MATLAB code is included during the packaging phase.

Prerequisites

1. Install and Configure the MATLAB Runtime
Complete this step only if you have not installed the MATLAB Runtime while installing the library.
2. Verify that you have a C++ compiler installed.
You can use the mbuild command within MATLAB to verify whether you have a C++ compiler installed. For more information, see mbuild.

Compiling C++ Driver Applications

To compile C++ Data Array driver applications at the system terminal, type:
mbuild driverApplication_mda.cpp
To compile C++ mwArray driver applications at the system terminal, type:
mbuild driverApplication_mwarray.cpp libSpGH.so
Note: You will need write permissions to the folder where you are compiling your driver application. If you do not have write permissions, copy the sample and generated component to a folder with write permissions, and then compile your driver application.

Executing C++ Driver Applications

At the system terminal, type:
./run_driver.sh %MATLAB_RUNTIME_INSTALL_LOCATION% driverApplication_mwarray
Replace %MATLAB_RUNTIME_INSTALL_LOCATION% with the installation folder for MATLAB Runtime.
Note: If executing a sample Data Array driver provided with your component, the compiled sample and the component's CTF must be collocated
